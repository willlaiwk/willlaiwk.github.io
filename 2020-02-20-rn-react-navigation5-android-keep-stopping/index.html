React Navigation 5.x 踩坑! Android keeps stopping 問題

February 20, 2020

...

一陣子沒碰 React Native，最近看到 react-navigation 已經進版到了 5.x，所以就參考了官方文件，並在 ios 與 android 模擬器上運行了一下，接著就撞牆了。

Android App 持續出現 keeps stopping

跑在 ios 上完全沒出錯，可是跑在 android 卻無法正常開啟 App，如下圖。

開始除錯

為了要知道發生什麼事，我在終端機(Terminal) 執行以下指令，查看模擬器的 log：

$ adb logcat *:E

這邊顯示的是 log 結果：

...
02-19 22:11:00.215  5550  5587 E AndroidRuntime: FATAL EXCEPTION: mqt_native_modules
02-19 22:11:00.215  5550  5587 E AndroidRuntime: Process: com.tsreactnativedemo, PID: 5550
02-19 22:11:00.215  5550  5587 E AndroidRuntime: java.lang.NoClassDefFoundError: Failed resolution of: Landroidx/swiperefreshlayout/widget/SwipeRefreshLayout;
...

我用上面 log 中第三段出現的 java.lang.NoClassDefFoundError 這段錯誤訊息內容去尋找資料，發現問題是在 react-native-screens 這個套件。

react-native-screens

這個套件目的是暴露 native 的 navigation 容器組件給 React Native 做使用。而 react-navigation 也相依於此，所以在 react-navigation 官方文件也說明要安裝此套件。

在 react-native-screens 的文件中寫到要在 android/app/build.gradle 中的 dependencies 區塊加入下面這段:

implementation 'androidx.appcompat:appcompat:1.1.0-rc01'
implementation 'androidx.swiperefreshlayout:swiperefreshlayout:1.1.0-alpha02'

加入後的完整的程式碼:

...
dependencies {
    implementation fileTree(dir: "libs", include: ["*.jar"])
    implementation "com.facebook.react:react-native:+" // From node_modules
    implementation 'androidx.appcompat:appcompat:1.1.0-rc01'
    implementation 'androidx.swiperefreshlayout:swiperefreshlayout:1.1.0-alpha02'
if (enableHermes) {
        def hermesPath = "../../node_modules/hermes-engine/android/";
        debugImplementation files(hermesPath + "hermes-debug.aar")
        releaseImplementation files(hermesPath + "hermes-release.aar")
    } else {
        implementation jscFlavor
    }
}
...

成功執行

在加入上面那段 code 後，就能順利在 Android 上執行了。

後記

解決問題的當天我回去看了 React Navigation 4.x 的文件，發現裡面有寫到 react-native-screens 的這段，但 React Navigation 5.x 的文件沒寫到。 而當我寫這篇文章時，React Navigation 5.x 的文件突然就加上了…，如果我晚點嘗試，是不是就不用踩這個坑了 XD…
